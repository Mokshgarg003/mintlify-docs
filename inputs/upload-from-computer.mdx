## [â€‹](#supported-file-types)Supported File Types

- **Images & video.** Screenshots, mockups, and visual references. Supported formats: PNG, JPG / JPEG, GIF, WebP, SVG, MP4, AVI
- **Documents**. PRDs, specs, and written context. Supported formats: PDF, DOCX / DOC, TXT, Markdown (.md)
- **Spreadsheets**. Data, analytics, and structured information. Supported formats: CSV, XLSX / XLS

## What to Upload

<AccordionGroup>
  <Accordion title="Product Screenshots">
    Capture your existing product for reference. Valuable screenshots:

    - Current screens you want to extend
    - Competitor interfaces for inspiration
    - Patterns you want to match
    - States you need to handle (empty, error, loading)
  </Accordion>
  <Accordion title="PRDs and Specs">
    Give Figr the requirements directly. Document content that helps:

    - Feature requirements
    - User stories
    - Acceptance criteria
    - Business constraints
    - Technical limitations
  </Accordion>
  <Accordion title="Analytics Data">
    Real data makes designs more realistic. Useful data files:

    - User behavior analytics (CSV exports)
    - Survey results
    - Engagement metrics
    - Conversion funnels
    - A/B test results
  </Accordion>
  <Accordion title="User Research">
    Insights that shape design decisions. Research files to share:

    - Interview transcripts
    - Usability test findings
    - Survey responses
    - Persona documents
    - Journey maps
  </Accordion>
</AccordionGroup>
