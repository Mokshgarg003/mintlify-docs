A strong Figr workflow usually looks like this:

1. [Set up product context](/product-intelligence/context-pods)
2. [Add product inputs](/input-research/input-guide)
3. [Set up design language](/product-intelligence/design-system)
4. [Ask for a UX review](/getting-started/tutorial/review)
5. [Map user flows and edge cases](getting-started/tutorial/map-edge-cases)
6. [Create product artifacts](/getting-started/tutorial/create-prds)
7. [Generate the prototype](/product-thinking/prototype)
8. [Refine with Select & Edit](getting-started/tutorial/high-fidelity-prototypes)
9. [Export, publish, or hand off](/export/options)

Use this workflow especially when working inside an existing product.
